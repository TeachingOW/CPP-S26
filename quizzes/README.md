2. **No Resources**: Try to complete without notes or internet
3. **Check Solutions**: Review detailed solutions after completion
4. **Identify Gaps**: Note topics where you struggled
5. **Review Materials**: Revisit relevant lecture notes and topics

1. **Start Early**: Begin review quizzes 1-2 weeks before exams
2. **Multiple Attempts**: Retake quizzes to reinforce learning

4. **Practice Coding**: Actually write and run code examples
5. **Study Groups**: Discuss tricky questions with classmates

## Question Types Explained

### 1. Multiple Choice (MC)
Select the best answer from 4 options. Focus on understanding concepts, not memorization.

**Example**:
```cpp
What is the output?
int x = 5;
cout << ++x;

A) 4  B) 5  C) 6  D) Compilation error
```

### 2. True/False (TF)
Determine if a statement is true or false. Watch for subtle wording.

### 3. Code Output Prediction
Predict exact program output. Pay attention to whitespace and newlines.

### 4. Bug Finding
Identify and fix bugs in provided code. Explain what was wrong.

### 5. Code Completion
Complete partial code to achieve specified functionality.

### 6. Short Answer
Explain concepts in 2-4 sentences. Be concise but complete.

### 7. Design Questions
Propose solutions to design problems. Focus on best practices.

## Study Tips and Strategies

### General Study Tips

💡 **Active Learning**:
- Write code by hand
- Trace through programs step-by-step
- Explain concepts to others
- Create your own examples

💡 **Practice**:
- Complete all homework assignments
- Work through tutorial problems
- Write code daily
- Experiment with variations

💡 **Understanding Over Memorization**:
- Focus on "why" not just "what"
- Understand common patterns
- Learn from mistakes
- Connect concepts together

### Topic-Specific Tips

**Pointers & Memory**:
- Draw memory diagrams
- Trace pointer operations
- Practice with dynamic allocation
- Understand stack vs heap

**OOP Concepts**:
- Design class hierarchies
- Understand virtual functions
- Practice with inheritance
- Master constructor/destructor order

**STL**:
- Know container complexities
- Practice with iterators
- Understand algorithm requirements
- Study common use cases

**Templates**:
- Start with simple examples
- Understand instantiation
- Practice type deduction
- Study specialization carefully

## Common Pitfalls

### ⚠️ Pointer Pitfalls
- Forgetting to initialize pointers
- Memory leaks with new/delete
- Dangling pointers
- Null pointer dereference

### ⚠️ OOP Pitfalls
- Forgetting virtual destructors
- Slicing objects
- Copy constructor issues
- Access specifier confusion

### ⚠️ STL Pitfalls
- Iterator invalidation
- Wrong container choice
- Modifying containers during iteration
- Forgetting to include headers

### ⚠️ Template Pitfalls
- Linker errors (implementation location)
- Complex error messages
- Type deduction issues
- Specialization vs overloading

## FAQ

**A**: Yes! All quizzes (except current week) are available for practice.
